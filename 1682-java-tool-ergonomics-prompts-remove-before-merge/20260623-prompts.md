## What `SchemaGeneratorTest` tests

It verifies that `SchemaGenerator` â€” a **compile-time** utility â€” correctly maps `javax.lang.model` types (the type system available inside annotation processors) to JSON Schema expressed as Java source code literals (`Map.of(...)` / `List.of(...)` strings).

## Testing mechanism: compilation-testing

The test can't just call `SchemaGenerator` directly because its API requires `TypeMirror`, `Types`, and `Elements` â€” objects that only exist during `javac` annotation processing. So it uses a **compilation-testing** pattern:

1. **`InMemorySource`** â€” wraps a Java source `String` as a `SimpleJavaFileObject`, so `javac` can compile it without writing source files to disk.

2. **`SchemaCapturingProcessor`** â€” a custom annotation processor (`@SupportedAnnotationTypes("*")`) that runs during compilation. Inside `process()`, it:
   - Scans each root element (class/record/enum/interface) for methods whose name starts with `schemaTarget` â€” then calls `generator.generateSchemaSource()` on that method's **return type**, recording `"methodName=schemaOutput"` in a static list.
   - For methods named `parametersTarget`, calls `generator.generateParametersSchemaSource()` on the method's parameter list.
   - For root types named `TestRecord*`, `TestEnum*`, or `TestSealed*`, generates schema for the type itself (records, enums, sealed interfaces).

3. **`compileAndCapture()`** / **`compileAndCaptureParams()`** â€” helper methods that:
   - Create an in-memory compilation unit from the source string
   - Invoke `JavaCompiler.getTask()` with a `StandardJavaFileManager` configured to write `.class` output to `target/test-schema-classes/`
   - Register `SchemaCapturingProcessor`
   - Call `task.call()` to trigger compilation â†’ annotation processing â†’ schema capture
   - Return the captured schema strings
