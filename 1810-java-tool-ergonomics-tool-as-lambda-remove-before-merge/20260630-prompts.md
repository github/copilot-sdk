### 17:20 Response

After the copilot code review agent has left comments on a PR, and I have addressed all the comments. What is the `gh` command to re-request review from Copilot?

```
gh pr edit <PR_NUMBER> -R <REPO> --add-reviewer "copilot-pull-request-reviewer"
```

That's the bot account name for the Copilot code review agent. Alternatively:

gh api --method POST "/repos/<OWNER>/<REPO>/pulls/<PR_NUMBER>/requested_reviewers" \
  -f "reviewers[]=copilot-pull-request-reviewer"
  
### 17:20 Prompt


But what if Copilot already is a reviewer, and I'm re-requesting review? Will that work?

Yes — requesting a reviewer that has already reviewed (or been requested) re-triggers the review. The API is idempotent in that sense: it moves the reviewer back to "pending review" state, which causes the Copilot review agent to run again.

So the same command works for both initial request and re-request.
