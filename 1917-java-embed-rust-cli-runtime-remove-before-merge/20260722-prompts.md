### 14:18 Prompt

Can the `maven-jar-plugin` be configured so that it creates a jar with only a `MANIFEST.MF` and nothing else? Can the contents of the manifest be specified in the configuration?

### 14:19 Response

Updated todo list

Yes to both.

**Empty JAR with only MANIFEST.MF:**

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-jar-plugin</artifactId>
    <configuration>
        <classesDirectory>${project.build.directory}/empty</classesDirectory>
        <includes>
            <include>**/*.nonexistent</include>
        </includes>
    </configuration>
</plugin>
```

Point `classesDirectory` at an empty directory (create it with `maven-antrun-plugin` or `maven-resources-plugin` in an earlier phase) and include nothing — the JAR contains only `META-INF/MANIFEST.MF`.

**Custom manifest content:**

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-jar-plugin</artifactId>
    <configuration>
        <archive>
            <manifestEntries>
                <Copilot-Runtime-Platform>darwin-arm64</Copilot-Runtime-Platform>
                <Copilot-Runtime-Version>${project.version}</Copilot-Runtime-Version>
                <Copilot-Runtime-Library>runtime.node</Copilot-Runtime-Library>
            </manifestEntries>
        </archive>
    </configuration>
</plugin>
```

The `<archive><manifestEntries>` block lets you add arbitrary key-value pairs to `MANIFEST.MF`. You can also use `<manifestFile>` to point at a pre-built manifest file instead.

**Why this matters for the plan:** The classifier JARs for `copilot-sdk-java-runtime` could be nearly-empty JARs containing just the `native/<classifier>/runtime.node` binary as a resource plus a manifest with platform metadata — no compiled classes needed. The `maven-jar-plugin` supports this directly.
